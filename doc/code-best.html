Design Proposal v2.0

时间管理局

时间是唯一货币，每一秒都在消耗你的生命

01 / Concept
核心设计理念

以电影《In Time》为灵感。黑为深渊，红为生命，线为骨架。去除一切冗余装饰，让线条本身成为美学语言。

01

时间即生命

登录即获得初始生命时长，倒计时从第一秒开始。所有界面设计围绕「流逝感」这一核心情感展开。

02

线条美学

拒绝填充式图标与色块装饰。所有图形元素均以等线宽 stroke 绘制，统一 1.25px 线宽，圆角端点，呼吸感留白。

03

紧迫而不焦虑

紧迫感通过倒计时动画与进度条收缩传递，而非杂乱的视觉噪音。界面本身保持极致克制。

04

多人共生

玩家之间可以发布任务、交易时间、组建联盟。社交属性与生存压力并存。

02 / Color
配色方案

三个色彩层级：深渊黑为基底，生命红为信号，暖白为信息。克制使用，让红色拥有最大冲击力。

07 / Features
功能模块

围绕「时间获取」与「时间消耗」构建完整经济循环。

生命倒计时

登录即开始，实时倒计时。归零则冻结，需特定方式复活。

系统任务

签到、浏览、学习、答题。低风险低收益，基础时间来源。

玩家悬赏

消耗时间发布任务，其他用户接取完成获取报酬。

时间交易

开放市场，用户直接交易时间。价格随供需浮动。

时间联盟

组建联盟，共享任务池，互助完成大型任务。

排行榜

最富有时长、最多交易、最活跃。前列获得系统奖励。

08 / Roadmap
开发路线图
PHASE 01

核心机制 — 第 1-4 周

注册登录、生命计时系统、基础 UI 框架、系统任务模块。确保核心循环跑通。

PHASE 02

社交系统 — 第 5-8 周

玩家任务发布与接取、任务审核、用户信用体系、消息通知。

PHASE 03

经济体系 — 第 9-12 周

时间交易市场、价格浮动、联盟系统、排行榜与成就。

PHASE 04

生态扩展 — 第 13+ 周

复活机制与道具、限时活动与赛季、数据看板、开放 API。
